---

## Padrão de Relatórios

### Quando Gerar

1. ✅ Após conclusão de uma fase de desenvolvimento
2. ✅ Após implementação de funcionalidades significativas
3. ✅ Após atualização de modelos de dados
4. ✅ Após criação/atualização de serviços
5. ✅ Após mudanças arquiteturais importantes

### Estrutura do Relatório

Todos os relatórios devem seguir o padrão estabelecido:

```
RELATORIO_EXECUCAO_XXX.md
```

**Seções Obrigatórias:**

1. **Informações Gerais**
   - Projeto, Repositório, Fase, Status

2. **Resumo Executivo**
   - Objetivos alcançados
   - Principais entregas

3. **Entregas Realizadas**
   - Detalhamento técnico
   - Status de cada item

4. **Métricas de Execução**
   - Cobertura de funcionalidades
   - Arquivos criados/modificados
   - Linhas de código

5. **Conformidade com Requisitos**
   - Verificação de cada requisito
   - Evidências de implementação

6. **Riscos e Desvios**
   - Riscos identificados
   - Desvios do planejado

7. **Próximas Etapas**
   - Roadmap
   - Prioridades

8. **Conclusão**
   - Resumo
   - Status para próxima fase

9. **Anexos**
   - Estrutura de arquivos
   - Exemplos de uso
   - Referências

### Numeração

- Relatório #001: Fase 1 - Modelagem e Interface Base
- Relatório #002: Fase 2 - Multi-Obra e Multi-Usuário
- Relatório #003: Fase 3 - (próxima fase)
- ...
